# juju-installer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Debian package (`juju-installer`) that enables `juju deploy postgresql` on a fresh Ubuntu 24.04+ with zero prior setup.

**Architecture:** A thin `/sbin/juju` wrapper script triggers a systemd socket-activated service to install snaps (root ops), then performs bootstrap/model-creation as the calling user (user ops). Mirrors the `lxd-installer` package already shipped in Ubuntu.

**Tech Stack:** POSIX shell (`/bin/sh`), systemd socket activation, debhelper packaging, python3 (for socket connection only)

**Reference:** The `lxd-installer` package on this system provides the proven pattern:
- `/sbin/lxc` — wrapper script
- `/usr/share/lxd-installer/lxd-installer-service` — service script
- `/usr/lib/systemd/system/lxd-installer.socket` — socket unit
- `/usr/lib/systemd/system/lxd-installer@.service` — service unit
- All maintainer scripts (postinst/prerm/postrm) are auto-generated by `dh_installsystemd`

**Design spec:** `docs/specs/2026-05-11-juju-installer-design.md`

### Task 3: Service Script (Root Operations)

**Files:**
- Create: `share/juju-installer-service`

This script runs as root via socket activation. It handles snap installation and LXD initialization only. It reads one byte from stdin (the mode), but currently treats all modes identically. All output goes to stdout (the socket) so the wrapper can display progress.

- [ ] **Step 1: Create the service script**

```sh
#!/bin/sh
set -eu

# Read mode byte from wrapper (i=install only, b=install+bootstrap-prep)
# Currently both modes are identical on the service side.
# Bootstrap runs in the wrapper as the calling user.
MODE=$(head -c1)

# Step 1: Install Juju snap if not present
if [ ! -x /snap/bin/juju ]; then
    echo "[1/3] Installing Juju snap..."
    snap install juju 1>&2
fi

# Step 2: Ensure LXD snap is available
if [ ! -x /snap/bin/lxd ]; then
    echo "[2/3] Installing LXD..."
    python3 -c 'import socket; s=socket.socket(socket.AF_UNIX); s.connect("/run/lxd-installer.socket"); s.send(b"x"); s.recv(1)' 1>&2
    i=0
    while [ "$i" -lt 90 ]; do
        sleep 1
        [ -x /snap/bin/lxd ] && break
        i=$((i + 1))
    done
    if [ ! -x /snap/bin/lxd ]; then
        echo "ERROR: LXD snap installation timed out." 1>&2
        exit 1
    fi
fi

# Step 3: Initialize LXD if needed
if ! /snap/bin/lxc storage list --format csv 2>/dev/null | grep -q default; then
    echo "[3/3] Initializing LXD..."
    /snap/bin/lxd init --auto 1>&2
fi

# Signal completion
echo 1
```

Write this to `share/juju-installer-service`.

- [ ] **Step 2: Make it not executable (will be run via `/bin/sh -eu`)**

The service unit runs it as `/bin/sh -eu /usr/share/juju-installer/juju-installer-service`, so the file itself does not need the execute bit. However, for local testing convenience, set it executable:

Run: `chmod +x share/juju-installer-service`

### Task 4: Wrapper Script

**Files:**
- Create: `sbin/juju`

This is the user-facing entry point at `/sbin/juju`. It handles four scenarios: (1) snap missing + non-deploy, (2) snap missing + deploy, (3) snap present + deploy + no controller, (4) everything ready (pass-through).

- [ ] **Step 1: Create the wrapper script**

```sh
#!/bin/sh
SNAP_BIN="/snap/bin/juju"

# --- Helper functions ---

show_progress() {
    printf "\r%-70s" "$1" >&2
}

clear_progress() {
    printf "\r%-70s\r" "" >&2
}

confirm_install() {
    printf "%s " "$1" >&2
    read -r answer < /dev/tty || answer="y"
    case "$answer" in
        [nN]*) echo "Aborted." >&2; exit 0 ;;
    esac
}

check_socket_access() {
    if [ ! -w "/run/juju-installer.socket" ]; then
        echo "Unable to trigger the installation of Juju." >&2
        echo "Please make sure you're a member of the 'lxd' system group." >&2
        echo "Run: sudo adduser \$USER lxd && newgrp lxd" >&2
        exit 1
    fi
}

trigger_service() {
    show_progress "[1/5] Installing Juju snap..."
    python3 -c 'import socket; s=socket.socket(socket.AF_UNIX); s.connect("/run/juju-installer.socket"); s.send(b"i"); s.recv(1)'
}

wait_for_snap() {
    i=0
    while [ "$i" -lt 90 ]; do
        sleep 1
        [ -x "${SNAP_BIN}" ] && return 0
        i=$((i + 1))
    done
    clear_progress
    echo "ERROR: Juju snap installation timed out." >&2
    exit 1
}

has_controller() {
    "${SNAP_BIN}" controllers --format json 2>/dev/null | grep -q '"lxd"'
}

has_model() {
    "${SNAP_BIN}" models --format json 2>/dev/null | grep -q '"welcome"'
}

do_bootstrap() {
    if ! has_controller; then
        show_progress "[4/5] Bootstrapping Juju controller (this may take a few minutes)..."
        if ! timeout 600 "${SNAP_BIN}" bootstrap lxd lxd >/dev/null 2>&1; then
            clear_progress
            echo "ERROR: Juju bootstrap failed or timed out. Re-run the command to retry." >&2
            exit 1
        fi
    fi

    if ! has_model; then
        show_progress "[5/5] Creating welcome model..."
        if ! timeout 60 "${SNAP_BIN}" add-model welcome >/dev/null 2>&1; then
            clear_progress
            echo "ERROR: Failed to create welcome model. Re-run the command to retry." >&2
            exit 1
        fi
    fi

    clear_progress
}

is_deploy_command() {
    [ "${1:-}" = "deploy" ]
}

# --- Main logic ---

if [ ! -x "${SNAP_BIN}" ]; then
    # Snap not installed — need to trigger service
    check_socket_access

    if is_deploy_command "$@"; then
        confirm_install "Would you like to install Juju snaps now (Y/n)?"
    else
        confirm_install "Would you like to install Juju snap now (Y/n)?"
    fi

    trigger_service
    wait_for_snap

    if is_deploy_command "$@"; then
        do_bootstrap
    fi
elif is_deploy_command "$@"; then
    # Snap installed but maybe no controller yet
    if ! has_controller || ! has_model; then
        do_bootstrap
    fi
fi

exec "${SNAP_BIN}" "$@"
```

Write this to `sbin/juju`.

- [ ] **Step 2: Make executable**

Run: `chmod +x sbin/juju`

- [ ] **Step 3: Run shellcheck**

Run: `shellcheck sbin/juju`

Fix any issues. Note: `$@` in `is_deploy_command "$@"` and `exec "${SNAP_BIN}" "$@"` is intentional and correct — shellcheck may warn about word splitting but the quoting is proper.

- [ ] **Step 4: Commit**

```bash

git commit -m "feat: add wrapper script for /sbin/juju"
```

These tests validate the wrapper script logic by mocking the filesystem and commands. They run without root and without actually installing anything.

- [ ] **Step 1: Create the test script**

```sh
#!/bin/sh
set -eu

PASS=0
FAIL=0
TOTAL=0

assert_equals() {
    TOTAL=$((TOTAL + 1))
    if [ "$1" = "$2" ]; then
        PASS=$((PASS + 1))
    else
        FAIL=$((FAIL + 1))
        echo "FAIL: $3"
        echo "  expected: $1"
        echo "  actual:   $2"
    fi
}

assert_contains() {
    TOTAL=$((TOTAL + 1))
    if echo "$2" | grep -q "$1"; then
        PASS=$((PASS + 1))
    else
        FAIL=$((FAIL + 1))
        echo "FAIL: $3"
        echo "  expected to contain: $1"
        echo "  actual: $2"
    fi
}

# --- Test: is_deploy_command detection ---

# Source just the function from the wrapper
is_deploy_command() {
    [ "${1:-}" = "deploy" ]
}

# Test: deploy is detected
is_deploy_command "deploy" && result="yes" || result="no"
assert_equals "yes" "$result" "is_deploy_command should detect 'deploy'"

# Test: status is not deploy
is_deploy_command "status" && result="yes" || result="no"
assert_equals "no" "$result" "is_deploy_command should not match 'status'"

# Test: help is not deploy
is_deploy_command "help" && result="yes" || result="no"
assert_equals "no" "$result" "is_deploy_command should not match 'help'"

# Test: empty args
is_deploy_command "" && result="yes" || result="no"
assert_equals "no" "$result" "is_deploy_command should handle empty string"

# Test: no args at all
is_deploy_command && result="yes" || result="no"
assert_equals "no" "$result" "is_deploy_command should handle no args"

# --- Test: show_progress output ---

show_progress() {
    printf "\r%-70s" "$1"
}

output=$(show_progress "[1/5] Installing Juju snap...")
assert_contains "Installing Juju snap" "$output" "show_progress should contain step text"

    assert_equals "yes" "yes" "snap binary exists on this system (scenario 4 pass-through)"

# --- Results ---
```
